日程及任务管理,好用的「滴答清单」

发表于 2019-11-10 | 分类于 生活 | | 阅读次数： | 阅读次数：

最近事情比较多，几年来根据个人习惯、偏好不断调整的日程管理（持续一整天或者固定时间进行）和任务管理（Todo）流程逐渐成熟，形成了两者相结合的效率体系，前阵子Byte.coffee的Slack听众群里有人发起过相关讨论，在此介绍一下我的方法：首先申明，我是不单独区分ToDo和日程的，我使用一款叫「滴答清单」的全平台管理软件（主要是iOS、Mac及网页端），同时是年付订阅会员。

零、方法论

GTD（Get things done）管理体系是目前市面上绝大部分效率管理类工具的基础，相关的书籍有《尽管去做》等，主要思想总结如下：

  1. 好记性不如烂笔头，尽量让脑袋处理事情（类比CPU），而非存储。
  2. 建立收集箱（box）概念，无论何时有任何想做的事项、灵感都放进这个箱子里。其实还是遵循2分钟原则，即如果短时间不能将待办事项归类、定时间、定优先级等，就先记录。
  3. 建立不同类别的容器箱，将box里的任务分类后逐一按照时间、优先级进行列表排列。
  4. 逐一check每项待办。

一、需求

我追求的是：

  1. 清楚地知道自己每天、每周需要干什么，不管是一整天的事宜还是只需短时间完成的小事项，只要是需要执行的都算。
  2. 能够方便地做计划，同时兼具回顾、概览功能；
  3. 能够在一个APP里搞定且全平台；
  4. 最好能够有习惯培养计数模块：打卡；
  5. 最好能够有看板模式，针对长时间项目进行跟踪管理。

前三项为满足基本心里预期，后两者是完全的加分项，Boom🎇升级到5.3版本的「滴答清单」全完能够满足所有需求。

二、使用情况

所有的想法都放在「收集箱」里，这个相当于资料库，无限灵活，可以说是能够心安理得地各种增减而内心没有一丝波澜。

日程处理

针对日程一类的事宜，「滴答清单」有个任务循环功能处理固定事项，循环条件设置丰富（包括固定时间重复多少次停止、截止到何时停止等）。

列表模式

该模式下，会列出当日所有需执行事项（配有单个任务优先级、分类、标签），一目了然。配合全局搜索与右上角筛选功能，非常便捷。

日历模式

该模式下有日视图（24小时时间线）、周视图、月视图3种，日视图、周视图将全天即没有设定具体时间只设定日期的任务置顶到界面顶部表示「全天进行」，其他任务按时间线填充。月视图为日历展示，能够清楚一览月度空闲情况（见封面图）。

看板模式（限电脑端）

这种模式非常适用于项目管理，我用来做阅读进度管理，细分为待读-正在读-读完三个子版面，将书籍添加进入看板后可以随意拖动切换状态，优先级根据紧急程度从红色到灰色排列显示，还有具体完成百分比图标显示，从此不再单独开备忘录写reading list。

打卡模块（限iOS端）

这是近期新功能，可选择不启用，相对独立，习惯设置打卡条件丰富（基本满足常见需求）且支持打卡评论。

聚焦「今天」页面

重要的展示功能模块「今天」，我每天打开这个模块，上面显示包括当日打卡（可不显示）、有定时提醒完成的任务、全天日程，同时还可切换到聚焦模式，神清气爽。

另外非常解决我个人使用痛点的一个功能——app图标右上角通知标记：显示每天要做的任务的总数，做一项少一项，而且还可以选择打卡习惯是否计入总数，非常贴心😋。此外还有番茄钟模块、历史统计页面等，个人不常用且在设置中选择隐藏，此处略。总之，「滴答清单」满足了我目前的所有需求，而且细节完善，妥妥地占据我所有设备Docker一席之地💜。
